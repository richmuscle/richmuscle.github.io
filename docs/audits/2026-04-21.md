## Findings

### P0 — Ship-blocking

**P0-1: writing.rs renders "Principal Platform Architect" in demoted writeup author lines**
- Tag: CONTENT
- File: `src/pages/writing.rs:280-282`
- Current: Three `.replace()` calls inject "Principal Platform Architect" into writeup bodies
- Impact: Contradicts positioning contract (no self-declared tier per CLAUDE.md) and staff-lens P1-4
- Fix: Replace all three with tier-neutral text (e.g., "Richard Mussell") or remove author line entirely. Audit `public/writeups/*.json` for embedded stale author stanzas.
- Patch agent: NOT spawned — content change requires human approval per working agreements

### P1 — Visible gap

**P1-1: CommandPalette creates disconnected signal on missing context**
- Tag: ENG
- File: `src/components/palette.rs:103-105`
- Current: `use_context::<GlobalAppState>().map(|s| s.palette_open).unwrap_or_else(|| create_rw_signal(false))`
- Impact: If GlobalAppState is absent, palette appears to work but is silently disconnected from Cmd+K toggle. Violates consolidation intent of ADR-003.
- Fix: Change to `use_context::<GlobalAppState>().expect("App provides GlobalAppState").palette_open` — consistent with layout.rs:15-16 pattern.

**P1-2: layout.rs unwrap() chains in reading progress scroll handler**
- Tag: ENG
- File: `src/components/layout.rs:25,28,29,31,32`
- Current: Five `unwrap()` calls chained on `web_sys::window()`, `.document()`, `.body()`, `.inner_height()` inside `#[cfg(not(feature = "ssr"))]` block
- Impact: Any None return panics the scroll handler, though practically safe in WASM browser context. Inconsistent with lib.rs:125-134 which uses `let Some(x) = ... else { return }`.
- Fix: Replace with Option chains (`and_then` or `let Some(...) else { return }`). Already done in lib.rs — pattern exists in codebase.

**P1-3: js_sys::eval() used for clipboard operations (7 locations)**
- Tag: SECURITY
- Files: `src/utils/browser.rs:60`, `src/pages/resume.rs:31,237`, `src/pages/one_pager.rs:29`, `src/pages/contact.rs:41`, `src/components/site_footer.rs:31`, `src/components/project.rs:129`
- Current: All use `js_sys::eval(&format!("navigator.clipboard.writeText({:?})..."))` pattern
- Impact: eval() is an unnecessary code-smell on a static site. All inputs are either compile-time constants (EMAIL) or escaped code snippets. Actual XSS risk is low.
- Fix: Replace with direct Clipboard API via `web_sys::Navigator::clipboard().write_text()` (requires adding `Clipboard` and `ClipboardEvent` to web-sys features). Extract to shared helper function.

**P1-4: Dead CSS rule `.about-pills-row`**
- Tag: ENG
- File: `style/pages/about.css`
- Current: Rule exists for Section 4 pills row removed in Stage 2.3A
- Fix: Delete the rule.

### P2 — Technical debt

**P2-1: GlobalAppState has 3 unwired portfolio_* fields**
- Tag: ENG
- Files: `src/state.rs:22-27`, `src/lib.rs:53-55`
- Current: `portfolio_category`, `portfolio_search`, `portfolio_index_tick` signals are created and provided but never consumed by any component.
- Fix: Either wire to home-page filtering (completing the SQLite search feature) or remove and relocate to local state when needed.

**P2-2: SQLite lifecycle lacks explicit cleanup documentation**
- Tag: ENG
- File: `src/db.rs:119`
- Current: `DB_PTR: AtomicUsize` stores raw `*mut sqlite3` pointer. Handle is opened but never explicitly closed. Cleanup is implicit on WASM module unload.
- Fix: Add doc comment to DB_PTR explaining lifecycle contract. Optionally implement explicit `sqlite3_close()` on `beforeunload` event.

**P2-3: 4 inner_html sinks render owner-authored JSON content without sanitization layer**
- Tag: SECURITY
- Files: `src/pages/writing.rs:284`, `src/components/project.rs:140`, `src/pages/project/detail.rs:458`, `src/pages/project/docs.rs:442`
- Current: All render `d.content` from static JSON files or `highlight_code()` output via `inner_html=`.
- Impact: Theoretical XSS if JSON files on CDN are compromised. Practical risk: near-zero (owner-authored static content).
- Fix: Document as accepted risk (static site, owner-controlled content). If defense-in-depth desired, add a lightweight HTML sanitizer before inner_html injection.

**P2-4: Missing Referrer-Policy and Permissions-Policy meta headers**
- Tag: SECURITY
- File: `index.html`
- Current: CSP and X-Frame-Options set. Referrer-Policy and Permissions-Policy absent.
- Fix: Add `<meta http-equiv="Referrer-Policy" content="strict-origin-when-cross-origin">` and Permissions-Policy restricting unused APIs. Low priority for static site.

**P2-5: db.rs feature gate boundary unclear in API surface**
- Tag: OPS
- File: `src/db.rs:336-378`, `src/lib.rs:9`
- Current: Public functions (`search_portfolio_projects`, `search_projects`, `sqlite_index_ready`) compile for all targets with internal cfg dispatch. This is correct behavior but the API surface doesn't clearly communicate feature availability.
- Fix: Add module-level documentation explaining that search functions always work (SQLite when available, in-memory fallback otherwise).
