### Unsafe Code Assessment (src/db.rs)

All 12 unsafe blocks assessed as **sound** under WASM's single-threaded execution model:

- `DB_PTR` static uses `AtomicUsize` for raw `*mut sqlite3` — correct for single-threaded WASM
- All `CString` conversions handle NUL termination properly
- All pointer dereferences check for null before use
- `sqlite3_finalize` cleanup paths are complete

**Recommendation:** Add `// SAFETY: Single-threaded WASM only` comment to `DB_PTR` static. No code changes needed for current deployment.

## Adversarial Critiques

### 1. Half-finished portfolio signal
**Critique:** Three of six registry projects are `Planned` with zero substance. A hiring manager sees one shipped project, two in-development, and three empty shells. This reads as "half-finished" not "deep."

**Neutralizing move:** Either (a) ship one more project to `Shipped`, (b) remove the three `Planned` entries from the public registry, or (c) add substantive design docs so they read as "we know what this needs" rather than "TBD."

### 2. Measured vs. unmeasured metrics
**Critique:** Security baseline audit claims "87/92 CIS controls passing" (measured). Identity-access says "<15ms (design target, unbenchmarked)." Observability says "-60% alert noise (design target, unbenchmarked)." A reader cannot distinguish built-and-measured from designed-but-hypothetical.

**Neutralizing move:** Move unbenchmarked targets into "Scope Boundaries" sections. Or run the benchmarks — WireGuard handshake latency and alert noise ratios are measurable.

### 3. Resume PDF friction
**Critique:** `/resume` links to a static PDF with no explanation. A recruiter expecting a printable page gets a file download. The GitHub Actions workflow verifies the PDF is >100KB but doesn't indicate which resume variant ships.

**Neutralizing move:** Add a note explaining the PDF is static, or implement HTML-to-PDF generation.
