**Detail:** GCP ACE and CKA listed as "In Progress" / "Target Q3-Q4 2026" with no enrollment evidence (exam registration, study timeline, course certificates). ### P2-2: Demo pages serve "Coming Soon" placeholders
**File:** `src/pages/project/demo.rs`
**Detail:** All four project demos (terraform-gcp, linux-admin-scripting, monitoring-observability, zero-trust-networking) render hardcoded "Coming Soon". For senior DevOps/infra roles, even a minimal interactive demo (architecture diagram, Terraform plan viewer) signals "I built this" vs "I read about it". Remove stubs or land real demos. Content P0 per ADR-005.

### P2-3: Work history lacks quantified impact metrics
**File:** `src/pages/resume.rs`
**Detail:** PISCES SOC internship claims activity counts (~50 alerts/day, ~3-5 incidents/week) but no outcomes (MTTTR reduction, false-positive rate improvement, detection rate). Duty-description vs impact narrative. Add: "Reduced false-positive rate from X% to Y% by tuning Z" or similar. Content P0 per ADR-005.

### P2-4: Sitemap references nonexistent project slugs
**File:** `public/sitemap.xml`
**Detail:** May still reference old slugs not in `get_infrastructure_fleet()`. Current projects: terraform-gcp, linux-admin-scripting, monitoring-observability, zero-trust-networking. SEO bots treat 404 sitemap entries as poor maintenance signal. Regenerate from `src/data/projects.rs`.

### P2-5: Resume PDF quality unverified
**File:** `public/pdfs/resume.pdf`
**Detail:** Deploy gate validates size (>100KB) but not content currency. Verify: dated 2026, includes latest experience, professional formatting, matches site narrative. Add "Last updated" badge on `resume.rs`.

### P2-6: Title consistency / positioning statement drift
**Files:** `src/data/mod.rs`, `index.html`, `src/pages/home.rs`, `src/pages/resume.rs`, `src/pages/writing.rs`
**Detail:** Title varies: "SOC-trained infrastructure engineer" (home hero), "Systems Administrator & DevOps Engineer" (resume), "Systems Engineering" (writing). Not contradictory, but lacks unified positioning statement. Recommend: single crisp narrative for all key pages.

### P2-7: Missing Referrer-Policy and Permissions-Policy meta headers
**File:** `index.html`
**Detail:** CSP and X-Frame-Options present. Referrer-Policy and Permissions-Policy absent. Low priority for static site, but adds credibility signal. GitHub Pages doesn't support response headers — must be inline meta tags. ~2 lines.

### P2-8: Five unwraps in layout.rs scroll closures
**File:** `src/components/layout.rs`
**Detail:** ReadingProgress scroll-listener unwraps window/document/body. In panic=abort release profile, unwrap failure tears down the entire WASM module. Replace with `if let Some(x) = ...` chains for graceful degradation. Cosmetic — elements always exist in browser, but idiomatic Rust prefers fallibility.
